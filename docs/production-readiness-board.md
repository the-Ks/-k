# 正式日常业务投用任务板

目标：让客服质检系统从“可运行 MVP”推进到“可被团队每天稳定使用”的状态。

## 0. 当前完成度

- 可演示：已完成。前端、后端、PostgreSQL、登录、角色菜单、消息、会话、质检、客户画像、BI、AI 质检链路均可运行。
- 内部试用：进行中。已补可迁移运行文档、mock 数据跟踪、最小测试脚本和数据库异常回退开关。
- 正式日常使用：进行中。还需要完成真实数据接入、权限细化、质检闭环、审计与运维能力。

## 1. 可迁移运行与发布

状态：已完成基础版。

验收标准：

- 新电脑 `git clone` 后执行 `npm install`、`npm test`、`npm run backend`、`npm run frontend` 可进入系统。
- 未配置数据库时可以使用 mock 数据预览。
- 配置 PostgreSQL 后可以执行 `npm run db:migrate:seed` 初始化数据库。
- 运行数据、PostgreSQL 二进制和本地 env 不进入 Git。

已完成：

- 修正 `.gitignore`，避免误忽略 `backend/src/data/mockData.js`。
- 新增 `npm test` 基线。
- 明确 `ALLOW_MOCK_FALLBACK`，生产默认不静默回退 mock。
- README 增加新电脑拉取运行步骤。

## 2. 真实数据接入

状态：待完成。

验收标准：

- 淘宝、微信导入字段有稳定映射表。
- 批量导入具备去重、失败重试、导入批次日志和错误明细。
- 图片 OCR、语音/视频转写、文件摘要以文本证据形式入库。
- 真实数据导入后不需要改前端即可查看消息、会话和质检结果。

建议任务：

- 补导入模板与字段字典。
- 增加导入失败明细 API。
- 增加重复导入幂等测试。
- 增加媒体证据解析状态筛选。

## 3. 权限与账号安全

状态：部分完成。

验收标准：

- 登录接口有失败限流。
- 账号支持禁用、重置密码、初始密码强制修改。
- 质检员只能访问授权部门、客服或会话范围。
- 客服只能查看本人会话、质检和客户画像。
- 关键权限规则有自动化测试覆盖。

建议任务：

- 增加登录失败计数与锁定窗口。
- 增加账号状态检查。
- 增加质检员数据范围模型。
- 增加权限负向测试。

## 4. 质检闭环

状态：部分完成。

验收标准：

- AI 建议分、客观分、人工最终分来源清晰。
- 人工调整必须填写原因。
- 每次调整写入 `operation_log`。
- 身份复核结果可持久化，不只停留在前端内存。
- 质检结果支持复核状态流转：待复核、已调整、已确认。

建议任务：

- 增加身份复核确认/驳回 API。
- 增加质检状态流转 API。
- 增加人工调整审计详情页。
- 增加按状态筛选质检记录。

## 5. AI 质检可靠性

状态：部分完成。

验收标准：

- AI 输出 schema 校验失败时不会更新最终分。
- AI 结果、prompt 版本、模型、输入快照、usage 和错误都可追溯。
- 多媒体消息没有文本证据时必须标记证据不足。
- 不同角色只能看到对应粒度的 AI 分析。

建议任务：

- 为三类 prompt 各补一组 schema 测试。
- 增加 AI 请求超时和 provider 错误展示。
- 增加 AI 结果重跑记录对比。
- 增加敏感信息脱敏检查。
